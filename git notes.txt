$ git push
fatal: The current branch master has no upstream branch.
To push the current branch and set the remote as upstream, use

    git push --set-upstream origin master

To have this happen automatically for branches without a tracking
upstream, see 'push.autoSetupRemote' in 'git help config'.


git push origin master-  
this is working as expected. Meaning: Push my local master branch to the remote named origin

git push-
Git asks internally: Where should I push current branch master?
Right now your local master branch is not linked/tracking any remote branch, so Git says:
fatal: The current branch master has no upstream branch.

Root cause- Right now my local master branch is not linked/tracking any remote branch, so Git says:

fatal: The current branch master has no upstream branch.

Fix- git push -u origin master
What does -u mean?
-u means set upstream.
It tells Git:
My local master branch should track origin/master
After this, Git remembers:
local master  →  origin/master


git push <remote-name> <local-branch>:<remote-branch>
git push origin master:master
git push origin master:master
Left side master = local branch.
Right side master = remote branch.
When only one master is written, Git assumes both names are same.


Before running:

```bash
git push -u origin master
```

the `.git/config` file had only the core repository settings and remote information:

```ini
[core]
	repositoryformatversion = 0
	filemode = false
	bare = false
	logallrefupdates = true
	symlinks = false
	ignorecase = true

[remote "origin"]
	url = https://github.com/projectrepo-rajnish/sampleRepo.git
	fetch = +refs/heads/*:refs/remotes/origin/*
```

At this point, Git knows that `origin` points to this GitHub repository:

```text
https://github.com/projectrepo-rajnish/sampleRepo.git
```

So this command works:

```bash
git push origin master
```

because we are explicitly telling Git:

```text
Push local master branch to remote origin.
```

But plain `git push` does not work yet because Git does not know which remote branch the local `master` branch should track.

After running:

```bash
git push -u origin master
```

Git adds this new section in `.git/config`:

```ini
[branch "master"]
	remote = origin
	merge = refs/heads/master
```

This means:

```text
The local branch master is now linked with the master branch on the remote named origin.
```

Now Git knows:

```text
local master  →  origin/master
```

So from next time, instead of writing:

```bash
git push origin master
```

we can simply write:

```bash
git push
```

Git will automatically understand that the current local branch `master` should be pushed to `origin/master`.

Similarly, for `git pull`, Git also knows that it should fetch from `origin` and bring changes from the remote `master` branch into the local `master` branch.
